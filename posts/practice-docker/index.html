最低限理解しておくべきDockerの概念

この記事のゴール

Dockerを使いこなせるようになる

  • Dockerの基本概念、image/network/volume を理解する
  • OSS等を試すときにDocker使ってサクッと試すことができるようになる
  • Dockerを使って開発環境を構築できるようになる

Dockerの基本概念

全体像

  • image
    • コンテナと呼ばれる実行環境のベースとなるもの
  • network
  • volume

以降でそれぞれについて詳しく見ていく。

Dockerイメージ

コンテナ起動に必要なファイルをまとめたファイルシステム。

  • OSのライブラリ、アプリケーションが含まれる
  • ex: webサーバー→ nginxやapacheのイメージ / rubyの実行環境→ rubyが含まれるイメージ を使用する
  • AUFSなどの特殊なファイルシステムが使用されていて階層構造。

Dockerイメージのファイルシステム

階層構造で管理されておりそれぞれの層をレイヤーと呼び、一種のコマンドを実行するたびにレイヤーは追加される。そしてこれらのレイヤーは読み取り専用である。

ex: nginxをイメージにインストールするコマンドで1レイヤーが作られ、設定ファイルをイメージ内にコピーするコマンドでレイヤーが追加される。

イメージからコンテナを起動したときに、読み書きが可能な層：コンテナレイヤーが作成される。
つまり、コンテナの起動 ＝ Dockerイメージ上に読み書き可能なレイヤを追加し指定されたプロセスを隔離された状態で実行すること

Dockerイメージの継承

あるイメージをベースにして、別のイメージを作成する事が可能。
ベースとなるイメージが同一であるイメージが複数ある場合は、ベースイメージ部分のレイヤーについては同じものを使用する＝ストレージの節約につながる

補足：Dockerイメージとコンテナの関係

イメージはあるコンテナのスナップショット。
コンテナを起動するというのは、既存のイメージをもとに新たなプロセスを開始するということ。

Images are frozen immutable snapshots of live containers. Containers are running (or stopped) instances of some image.
